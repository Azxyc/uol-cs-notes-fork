## Building New Relations from Given Ones
### Inverse Relation
Given a realtion {% raw %}<pre>\(R\subseteq A \times B\)</pre>{% endraw %}. We define the inverse relation {% raw %}<pre>\(R^{-1}\subset B\times A\)</pre>{% endraw %} by:

{% raw %}<pre>\[R^{-1}=\{(b,a)\vert (a,b) \in R\}\]</pre>{% endraw %}

Example:

* The inverse of the relation *is a parent of* on the set of people is the relation *is a child of*.

In other words if you swap the elements of a given relation you should get the inverse relation.

#### Example
{% raw %}<pre>\(A=\{1,2,3,4\},R=\{(x,y)\vert x\leq y\}\)</pre>{% endraw %}

Therefore: 

{% raw %}<pre>\(R=\{(1,1),(1,2),(1,3),(1,4),(2,2),(2,3),(2,4),(3,3),(3,4),(4,4)\}\)</pre>{% endraw %}

And:

{% raw %}<pre>\(R=\{(1,1),(2,1),(3,1),(4,1),(2,2),(3,2),(4,2),(3,3),(4,3),(4,4)\}\)</pre>{% endraw %}

You could also say:

{% raw %}<pre>\(R^{-1}=\{(y,x)\vert x \leq y\} = \{(u,v)\vert u\geq v\}\)</pre>{% endraw %}

In these examples you either swap the predicate to denote the inverse or you swap the evaluation such that it produces the inverse.

### Composition of Relations
Let {% raw %}<pre>\(R\subseteq A\times b\)</pre>{% endraw %} and {% raw %}<pre>\(s\subseteq B\times C\)</pre>{% endraw %}. The (functional) composition of {% raw %}<pre>\(R\)</pre>{% endraw %} and {% raw %}<pre>\(S\)</pre>{% endraw %}, denoted by {% raw %}<pre>\(S\circ R\)</pre>{% endraw %}, is the binary relation between {% raw %}<pre>\(A\)</pre>{% endraw %} and {% raw %}<pre>\(C\)</pre>{% endraw %} given by:

{% raw %}<pre>\[S\circ R = \{(a,c)\vert \text{ exists } b\in B \text{ such that } aRb \text{ and } bSc\}\]</pre>{% endraw %}

The notation {% raw %}<pre>\(aRb\)</pre>{% endraw %} is another way of writing {% raw %}<pre>\((a,b)\in R\)</pre>{% endraw %}.
{:.info}

```mermaid
graph LR
subgraph A
a[1]
end
subgraph B
b[ ]
end
subgraph C
c[2]
end 
a -->|R subset A * B| b
b -->|S subset B * C| c
```

Example:

* If {% raw %}<pre>\(R\)</pre>{% endraw %} is the relation *is a sister of* and {% raw %}<pre>\(S\)</pre>{% endraw %} is the relation *is a parent of* then:
	* {% raw %}<pre>\(S\circ R\)</pre>{% endraw %} is the relation *is an aunt of*.
	* {% raw %}<pre>\(S\circ S\)</pre>{% endraw %} is the relation *is a grandparent of*.
	
#### Example

* {% raw %}<pre>\(R:\)</pre>{% endraw %} is a sister of
* {% raw %}<pre>\(S:\)</pre>{% endraw %} is a parent of
* {% raw %}<pre>\(S\circ R=\{(a,c)\vert\text{ exists } b\in B\text{ such that } aRb \text{ and } bSc\}\)</pre>{% endraw %}

```mermaid
graph TD
fm[Fred and Mavis] --- Alice
fm --- ks[Ken and Sue]
ks --- Jane
ks --- Fiona
ks --- Alan
jm[John and Mary] --- ks
jm --- Mike
jm --- Penny
```

* Alice {% raw %}<pre>\(R\)</pre>{% endraw %} Ken and Ken {% raw %}<pre>\(S\)</pre>{% endraw %} Alan so Alice {% raw %}<pre>\(S\circ R\)</pre>{% endraw %} Alan.
	* This can also be written as {% raw %}<pre>\((\text{Alice, Alan})\in S\circ R\)</pre>{% endraw %}

## Computer Friendly Representation of Binary Relations - Matrices

Let {% raw %}<pre>\(A=\{a_1,\ldots,a_n\},B=\{b_1,\ldots,b_m\}\)</pre>{% endraw %} and {% raw %}<pre>\(R\subseteq A\times B\)</pre>{% endraw %}.

We represent {% raw %}<pre>\(R\)</pre>{% endraw %} by an array {% raw %}<pre>\(M\)</pre>{% endraw %} of {% raw %}<pre>\(n\)</pre>{% endraw %} rows and {% raw %}<pre>\(m\)</pre>{% endraw %} columns. Such an array is called an {% raw %}<pre>\(n\)</pre>{% endraw %} by {% raw %}<pre>\(m\)</pre>{% endraw %} matrix.

The entry in row {% raw %}<pre>\(i\)</pre>{% endraw %} and column {% raw %}<pre>\(j\)</pre>{% endraw %} of this matrix is given by {% raw %}<pre>\(M(i,j)\)</pre>{% endraw %} where:

{% raw %}<pre>\[
M(i,j)=\begin{cases}
1 & \text{ if } (a_i,b_j)\in R\\
0 & \text{ if } (a_1,b_j)\notin R
\end{cases}
\]</pre>{% endraw %}

### Example 1
Let {% raw %}<pre>\(A=\{1,3,5,7\}, B=\{2,4,6\}\)</pre>{% endraw %} and:

{% raw %}<pre>\[U=\{(x,y)\in A\times B\vert x + y = 9\}\]</pre>{% endraw %}

Assume an enumeration {% raw %}<pre>\(a_1=1,a_2=3,a_3=5,a_4=7\)</pre>{% endraw %} and {% raw %}<pre>\(b_1=2,b_2=4,b_3=6\)</pre>{% endraw %}. Then {% raw %}<pre>\(M\)</pre>{% endraw %} represents {% raw %}<pre>\(U\)</pre>{% endraw %}, where:

{% raw %}<pre>\[
M = \begin{bmatrix}
0 & 0 & 0\\
0 & 0 & 1\\
0 & 1 & 0\\
1 & 0 & 0
\end{bmatrix}
\]</pre>{% endraw %}

When representing in a matrix the rows are the items in set {% raw %}<pre>\(A\)</pre>{% endraw %} going down and the columns are the items in set {% raw %}<pre>\(B\)</pre>{% endraw %} going across.
{:.info}

You can then read the answers from the matrix as: {% raw %}<pre>\(U=\{(7,2),(5,4),(4,6)\}\)</pre>{% endraw %}.

{% raw %}<pre>\[
R: \begin{bmatrix}

\end{bmatrix}
S: \begin{bmatrix}

1&0
\end{bmatrix}

Calculate the binary relation matrix of {% raw %}<pre>\(S\circ R\)</pre>{% endraw %}:

If you transpose the row {% raw %}<pre>\(a\)</pre>{% endraw %} in the matrix {% raw %}<pre>\(R\)</pre>{% endraw %} on the column {% raw %}<pre>\(x\)</pre>{% endraw %} in the matrix {% raw %}<pre>\(S\)</pre>{% endraw %} you can compare to see of {% raw %}<pre>\(a\)</pre>{% endraw %} is a subset of {% raw %}<pre>\(y\)</pre>{% endraw %}. If it is then you put a 1 in the resultant matrix and if not you put a zero:

{% raw %}<pre>\[
S\circ R:\begin{bmatrix}
1&1\\
1&0
\end{bmatrix}
\]</pre>{% endraw %}

#### Boolean Matrix Product
Given two matrices with entries 1 and 0 representing the relations we can form the matrix representing the composition. This is called the logical (Boolean) matrix product.

Let {% raw %}<pre>\(A=\{a_1,\ldots,a_n\},B=\{b_1,\ldots,b_m\}\)</pre>{% endraw %} and {% raw %}<pre>\(C=\{c_1,\ldots,c_p\}\)</pre>{% endraw %}.

The logical matrix {% raw %}<pre>\(M\)</pre>{% endraw %} representing {% raw %}<pre>\(R\)</pre>{% endraw %} is given by:

{% raw %}<pre>\[
M(i,j)=\begin{cases}
1 & \text{ if } (a_i,b_j)\in R\\
0 & \text{ if } (a_1,b_j)\notin R
\end{cases}
\]</pre>{% endraw %}

The logical matrix {% raw %}<pre>\(N\)</pre>{% endraw %} representing {% raw %}<pre>\(S\)</pre>{% endraw %} is given by:

{% raw %}<pre>\[
N(i,j)=\begin{cases}
1 & \text{ if } (b_i,c_j)\in S\\
0 & \text{ if } (b_1,c_j)\notin S
\end{cases}
\]</pre>{% endraw %}

Then the entries {% raw %}<pre>\(P(i,)\)</pre>{% endraw %} of the logical matrix {% raw %}<pre>\(P\)</pre>{% endraw %} representing {% raw %}<pre>\(S\circ R\)</pre>{% endraw %} are given by:

* {% raw %}<pre>\(P(i,j)=1\)</pre>{% endraw %} if there existsw {% raw %}<pre>\(l\)</pre>{% endraw %} with {% raw %}<pre>\(1\leq l\leq m\)</pre>{% endraw %} such that {% raw %}<pre>\(M(i,l)=1\)</pre>{% endraw %} and {% raw %}<pre>\(N(i,j)=1\)</pre>{% endraw %}.
* {% raw %}<pre>\(P(i,j)=0\)</pre>{% endraw %}, otherwise.

This is the same as a product of matrices, {% raw %}<pre>\(P=MN\)</pre>{% endraw %}. Instead of addition and multiplication we use logical OR and AND.
