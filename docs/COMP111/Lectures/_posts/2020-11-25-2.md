## Marginalisation
Given an joint distribution {% raw %}<pre>\(\mathbf{P}(F_1,\ldots,F_k)\)</pre>{% endraw %}, one can compute the **marginal** probabilities of the random variables {% raw %}<pre>\(F_i\)</pre>{% endraw %} by summing out the remaining variables.

For example:

{% raw %}<pre>\[P(\text{Cavity}=1)=0.108+0.012+0.072+0.008=0.2\]</pre>{% endraw %}

is the sum of the entries in the first row of:

| | {% raw %}<pre>\(\text{Toothache}=1\)</pre>{% endraw %} | {% raw %}<pre>\(\text{Toothache}=1\)</pre>{% endraw %} | {% raw %}<pre>\(\text{Toothache}=0\)</pre>{% endraw %} | {% raw %}<pre>\(\text{Toothache}=0\)</pre>{% endraw %} |
| :-: | :-: | :-: | :-: | :-: |
| | {% raw %}<pre>\(\text{Catch}=1\)</pre>{% endraw %} |  {% raw %}<pre>\(\text{Catch}=0\)</pre>{% endraw %} |  {% raw %}<pre>\(\text{Catch}=1\)</pre>{% endraw %} |  {% raw %}<pre>\(\text{Catch}=0\)</pre>{% endraw %} | 
| {% raw %}<pre>\(\text{Cavity}=1\)</pre>{% endraw %} | 0.108 | 0.012 | 0.072 | 0.008 |
| {% raw %}<pre>\(\text{Cavity}=0\)</pre>{% endraw %} | 0.016 | 0.064 | 0.144 | 0.576 |

## Conditional Distributions
We can also computer conditional distributions from the full joint distributions.

* We use the {% raw %}<pre>\(\mathbf{P}\)</pre>{% endraw %} notion for conditional distributions.
* {% raw %}<pre>\(\mathbf{P}(F\ \vert\  G)\)</pre>{% endraw %} gives the **conditional distribution of {% raw %}<pre>\(F\)</pre>{% endraw %} given {% raw %}<pre>\(G\)</pre>{% endraw %}** given by the probabilities {% raw %}<pre>\(P(F=r\ \vert\  G=s)\)</pre>{% endraw %} for all values {% raw %}<pre>\(r\)</pre>{% endraw %} and {% raw %}<pre>\(s\)</pre>{% endraw %}.
* Using {% raw %}<pre>\(\mathbf{P}\)</pre>{% endraw %} notation, the general version of the product rule is as follows:

{% raw %}<pre>\[\mathbf{P}(F,G)=\mathbf{P}(F\ \vert\ G)\mathbf{P}(G)\]</pre>{% endraw %}

This stands for the list of equations:

{% raw %}<pre>\[
\begin{aligned}
P(F=r_1,G=s_1)&=P(F=r_1\ \vert\ G=s_1)P(G=s_1)\\
P(F=r_2,G=s_2)&=P(F=r_2\ \vert\ G=s_2)P(G=s_2)\\
\ldots&=\ldots
\end{aligned}
\]</pre>{% endraw %}

## Probabilistic Inference
Probabilistic inference can be characterised as the computation of posterior probabilities:

{% raw %}<pre>\[\mathbf{P}(Q\ \vert\ E_1=e_1,\ldots,E_n=e_n)\]</pre>{% endraw %}

for query variable {% raw %}<pre>\(Q\)</pre>{% endraw %} given observed evidence {% raw %}<pre>\(e_1,\ldots,e_n\)</pre>{% endraw %}.

In principle, we can use the full joint distribution to do this.

### Example - {% raw %}<pre>\(\mathbf{P}(\text{Cavity}\ \vert\ \text{Toothache}=1)\)</pre>{% endraw %}
We want to compute the conditional probability distribution for {% raw %}<pre>\(\text{Cavity}\)</pre>{% endraw %} given the observation or evidence of {% raw %}<pre>\(\text{Toothache}=1\)</pre>{% endraw %}.

Thus we want to compute:

* {% raw %}<pre>\(P(\text{Cavity}=1\ \vert\ \text{Toothache}=1)\)</pre>{% endraw %}
* {% raw %}<pre>\(P(\text{Cavity}=0\ \vert\ \text{Toothache}=1)\)</pre>{% endraw %}

We can easily obtain this using the table:

{% raw %}<pre>\[
\begin{aligned}
P(\text{Cavity}=1\ \vert\ \text{Toothache}=1)&=\frac{P(\text{Cavity}=1, \text{Toothache}=1)}{P(\text{Toothache}=1)}\\
&=\frac{0.12}{0.2}=0.6\\
\end{aligned}
\]</pre>{% endraw %}

{% raw %}<pre>\[
\begin{aligned}
P(\text{Cavity}=0\ \vert\ \text{Toothache}=1)&=\frac{P(\text{Cavity}=0, \text{Toothache}=1)}{P(\text{Toothache}=1)}\\
&=\frac{0.08}{0.2}=0.4
\end{aligned}
\]</pre>{% endraw %}

The denominator 0.2 can be viewed as a **normalisation constant** {% raw %}<pre>\(\frac{1}{\alpha}=5\)</pre>{% endraw %} for the distribution {% raw %}<pre>\(\mathbf{P}(\text{Cavity}\ \vert\ \text{Toothache}=1)\)</pre>{% endraw %}, ensuring that is adds up to 1.

This means that instead of completing the computation above we can consider this:

{% raw %}<pre>\[
\begin{aligned}
\mathbf{P}(\text{Cavity}\ \vert\ \text{Toothache}=1)&=\alpha\mathbf{P}(\text{Cavity, Toothache}=1)\\
&= \alpha(0.12,0.08)\\
&=5(0.12,0.08)\\
&= (0.6,0.4)
\end{aligned}
\]</pre>{% endraw %}

In this case {% raw %}<pre>\(\alpha\)</pre>{% endraw %} is defined as a value that makes the sum of the vector 1. In this case it is 5.
{:.info}

### Combinatorial Explosion
This approach does not scale well: for a domain described by {% raw %}<pre>\(n\)</pre>{% endraw %} random variable taking {% raw %}<pre>\(k\)</pre>{% endraw %} distinct values each we face tow problems:

* Writing up the full joint distribution requires {% raw %}<pre>\(k^n-1\)</pre>{% endraw %} entries.
	* The {% raw %}<pre>\(-1\)</pre>{% endraw %} in this case is as the full joint distribution adds to 1. To optimise we can not calculate the last value and just take the difference of the existing values and one.
	* There are too many calculations
* How to we find the numbers (probabilities) for the entries?
	* We don't have enough raw data.

For these reasons, the full joint distribution in tabular form in not a practical tool for building reasoning systems.
